Maximum Security PAT Guide

1

Study Security Requirements

Before creating a token, understand these critical security points:

  • Tokens are like passwords - protect them accordingly
  • RepoWave does NOT store tokens anywhere
  • You MUST delete tokens from GitHub after use
  • Set expiration to 1 day maximum
  • Select minimum required permissions only
2

Navigate to GitHub Settings

Log in to your GitHub account. Click on your profile picture in the top-right corner and select Settings from the dropdown menu.

3

Access Developer Settings

In the left sidebar, scroll down and click on Developer settings (located at the bottom of the settings menu).

4

Select Personal Access Tokens

Click on Personal access tokens, then choose Tokens (classic) from the submenu options.

5

Generate New Token

Click the Generate new token button, then select Generate new token (classic).

You may be prompted to enter your GitHub password for security verification.

6

Configure Maximum Security Settings

Token name: "RepoWave - Delete After Use"

Expiration: 1 day (Maximum Security)

Description: "For batch repository operations - DELETE IMMEDIATELY AFTER USE"

7

Select Minimum Required Scopes

Check ONLY these permissions:

  • repo - Full control of private repositories (required)
  • admin:org - Only if managing organization repositories
Security Best Practice:

Never select more permissions than absolutely necessary. Uncheck all other boxes.

8

Generate and Copy Token

Click Generate token at the bottom of the page.

Your token will appear only once. Copy it immediately to your clipboard.

Critical: Delete Token After Use

You MUST delete your token from GitHub immediately after using RepoWave:

  1. Return to GitHub Settings ‚Üí Developer settings ‚Üí Personal access tokens
  2. Find your token in the list
  3. Click the trash can icon to delete it
MANDATORY SECURITY STEP:

RepoWave does NOT store tokens. The token remains active on GitHub until YOU delete it. Failure to delete exposes your account to security risks.
